    Documentation System¶

    This project's documentation is automatically generated by doc-assistant — a Claude-powered documentation tool that runs on every git push.

    What runs on every push¶

    Task What it produces
    Technical docs docs/ — architecture, API reference, configuration
    User guides docs/guides/ — getting started, usage, deployment
    Changelog CHANGELOG.md — version history from git commits
    HTML site site/index.html — this navigable site

    All tasks run in the background so your push completes immediately. Progress is logged to .doc-assistant.log at the project root.

    Running manually¶

    # From the project root
python ~/Projects/doc-assistant/main.py docs      .
python ~/Projects/doc-assistant/main.py guide     .
python ~/Projects/doc-assistant/main.py changelog .

# Rebuild the HTML site
python ~/Projects/doc-assistant/build_docs.py .

    Adding screenshots¶

    Place screenshots in docs/assets/screenshots/ and embed them in any page:

    ![Login screen](../assets/screenshots/login.png)

    See docs/assets/screenshots/README.md for naming conventions and format guidelines.

    Customising the documentation¶

    File Purpose
    mkdocs.yml Site structure, theme, navigation
    ~/Projects/doc-assistant/prompts/docs.md System prompt for technical docs
    ~/Projects/doc-assistant/prompts/guides.md System prompt for user guides
    ~/Projects/doc-assistant/prompts/changelog.md System prompt for changelog

    Viewing the site locally¶

    # Serve with live reload (from project root)
mkdocs serve

# Or just open the built output directly
start site/index.html

    Source¶

    doc-assistant lives at ~/Projects/doc-assistant/. Its own documentation is at ~/Projects/doc-assistant/site/index.html.
